Introduction

In this project, we have provided a sketch code which contains there mapreduce jobs as following:

  1. CreateGraph (done): add one column, initial pagerank value, to the input pagerank adjacency matrix (AM). Then, we pass it to the PageRank program to calculate the pagerank values.
  2. PageRank (your implementation): take the transformed AM matrix and calculate pagerank values for all pages.
  3. CleanupResults: remove the targetUrls column and output <sourceUrl, pagerank value> as final result.

The detail dataflow could be seen in figure 1. Part 1 and Part 3 is given as full solution in this pipeline, you will implement the 2nd part of PageRank program.

Figure 1. Hadoop PageRank dataflow

Normally, for any Hadoop MapReduce program, input data are uploaded and stored in the Hadoop Distributed File System (HDFS) before computation in order to generate <key,value> pairs to the mapper. Initially, the PageRank input data is stored in the format of adjacency matrix as a file(s) in the local file system. Then, it will be uploaded to the HDFS and is distributed across the compute nodes. Hadoop framework read the application records from HDFS with the InputFormat interface, and generate <key, value> pairs input stream. Each Map function produces zero or more intermediate (key, value) pairs, by consuming one input (key, value) pair. For this PageRank program, the map function applies the calculation PR(v) / L(V) to each <key, value> pair, where the key is the unique id or name of web page, and the value contains the current rank value of web page and its out link information. Map tasks then generate intermediate (key, value) pairs, whose value is the partial rank value of web pages. Each reduce task aggregates all the partial value of specific web page by applying the Equ.2. The aggregated global rank values is write back to HDFS, which in turns is used as input in next iterations (if any).
